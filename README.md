# AI-ассистент браузера (Chrome Extension + ML Backend)

Интеллектуальный браузерный помощник с анимированным персонажем **Yumi**, который управляет реальным браузером через команды на естественном языке. Решает задачу автоматизации рутинных действий пользователя в интернете: поиск товаров, сравнение цен на маркетплейсах, получение актуальной информации — всё через простой текстовый запрос.

---

## Реализованная функциональность

- Управление браузером на естественном языке (навигация, клики, ввод текста, скролл);
- Поиск и сравнение цен на маркетплейсах (Wildberries, Ozon, Avito);
- Быстрый поиск фактов и информации через DuckDuckGo + Jina Reader без открытия браузера;
- Получение актуальных курсов валют (ЦБ РФ);
- Пошаговый агентный цикл с принятием решений на каждом шаге (LLM-агент);
- Визуальный браузинг со скриншотами через Playwright;
- Стриминг результатов в реальном времени (Server-Sent Events);
- Ручной режим — возможность задать JSON-план действий и выполнить его напрямую;
- Анимированный персонаж Yumi с 12 эмоциональными состояниями.

## Особенности проекта

- **Умная маршрутизация запросов** — система автоматически выбирает оптимальный путь: быстрый ответ без браузера, маркетплейс-парсинг или визуальный агент со скриншотами;
- **Анимированный персонаж Yumi** — anime-маскот с эмоциями (радость, задумчивость, ошибка и др.), который делает взаимодействие живым и дружелюбным;
- **Работа с реальным браузером** — не эмуляция, а настоящее управление вкладками Chrome через Content Scripts + Playwright MCP.

---

## Основной стек технологий

**Frontend (Chrome Extension):**
- HTML5, CSS3, JavaScript (ES6+)
- Chrome Extension Manifest V3
- Chrome Side Panel API, Messaging API, Scripting API

**Backend (ML Service):**
- Python 3.12, FastAPI, Uvicorn
- Playwright MCP (Model Context Protocol) — автоматизация браузера
- OpenAI-совместимый клиент (ZvenoAI) — LLM-инференс (Gemini 2.0 Flash, Qwen 3)
- BeautifulSoup4 — парсинг HTML
- httpx — асинхронные HTTP-запросы
- SSE (Server-Sent Events) — стриминг результатов

**Инфраструктура:**
- Docker, Docker Compose
- Git

## Среда запуска

- ОС: Windows 10+, macOS, Linux (debian-like для Docker)
- Браузер: Google Chrome (для расширения)
- Docker и Docker Compose (для бэкенда)
- Либо Python 3.12+ и Node.js 20+ (для запуска без Docker)
- API-ключ ZvenoAI (для LLM-инференса)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd Hackathon_mts
```

### 2. Настройка переменных окружения

```bash
cp ml-service/.env.example ml-service/.env
```

Отредактируйте `ml-service/.env`:

```
QWEN_API_KEY=ваш_ключ_zveno_ai
BACKEND_URL=http://localhost:8080
```

### 3. Запуск бэкенда

**Вариант A — Docker (рекомендуется):**

```bash
docker compose up --build
```

ML-сервис будет доступен на `http://localhost:8001`.

**Вариант B — без Docker:**

```bash
cd ml-service
pip install -r requirements.txt
uvicorn ml_service:app --host 0.0.0.0 --port 8001
```

### 4. Установка Chrome-расширения

1. Откройте `chrome://extensions` в браузере
2. Включите **Режим разработчика** (Developer mode)
3. Нажмите **Загрузить распакованное** (Load unpacked)
4. Выберите папку `frontend/`
5. Расширение появится в панели — нажмите на иконку для открытия Side Panel

## Как пользоваться

### Сценарий 1 — Поиск товаров на маркетплейсе

1. Откройте боковую панель расширения
2. Введите: `Найди кроссовки Nike до 5000 рублей на Wildberries`
3. Агент откроет Wildberries, выполнит поиск, отфильтрует по цене и вернёт результаты

### Сценарий 2 — Быстрый вопрос

1. Введите: `Какой курс доллара сегодня?`
2. Система получит данные с ЦБ РФ и ответит мгновенно (без открытия браузера)

### Сценарий 3 — Произвольная автоматизация

1. Введите: `Открой Wikipedia и найди статью про машинное обучение`
2. Агент пошагово выполнит действия в реальной вкладке браузера

### Сценарий 4 — Ручной JSON-план

Вставьте в поле "План" массив действий:

```json
[
  { "type": "navigate", "url": "https://example.com" },
  { "type": "click", "selector": "a.link" },
  { "type": "type", "selector": "input[name=q]", "text": "hello", "clear": true }
]
```

## API-эндпоинты бэкенда

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/query` | Отправить запрос на обработку (возвращает `request_id`) |
| GET | `/api/result/{id}` | Получить результат по ID |
| GET | `/api/stream/{id}` | SSE-стрим выполнения в реальном времени |
| POST | `/api/step` | Один шаг агента (для пошагового режима) |
| POST | `/api/plan` | Генерация плана действий по запросу |
| GET | `/api/health` | Проверка состояния сервиса |
